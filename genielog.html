Questions / Réponses + debug

  • Si, au moment de faire la gestion du conflit, git vous indique qu'il n'y à pas de conflit, effectuer les commandes ci-dessous
    • Ce bug est dû au fait qu'une récente mise à jour de git a modifié le comportement du -a dans git commit.
    • Dans les anciennes versions, le -a ajoutait tout les fichiers au commit. Dans les nouvelles versions, -a ajoute uniquement les fichiers ayant déjà été commités une fois...
git checkout conflict echo "je suis un conflit" >> conflit.txt git add . git commit -m "conflit from conflict!" git checkout master echo "Mais non y'a pas de conflit ici" >> conflit.txt git add . git commit -m "conflit from master !" git merge conflict // Vous pouvez reprendre le TP !
  • A la dernière étape, le fichier test3.txt n'apparait pas dans le dossier Git/, mais j'ai bien la modification dans git log
    • => En fait, Git est resté bloqué sur l'idée que vous vouliez supprimer test3.txt du dossier Git/ (en raison d'une commande effectuée beaucoup plus haut)
    • Vous pouvez faire un git stash pour le voir. cette commande vous permet de dire à Git de mettre les modifications en cours de côté, et d'afficher la dernière version du dépôt connue.
    • Bravo, vous venez de finir le TP ;-)
  • A la dernière étape, j'ai une erreur : Git refuse de faire le push !
    • Vous avez très probablement mal tapé une commande dans le dossier Git/ : git config receive.denyCurrentBranch ignore
    • Copiez/collez cette ligne et réessayez ! :-)

> Ressources / Documents & infos utiles

  • Si vous utilisez un dépôt Git distant :
    • Le plus simple est de créer un projet sur le serveur Git, puis de le cloner : git clone
    • Il faut penser à faire des git push pour envoyer les informations sur le serveur, et des git pull pour les récupérer !
      • git pull origin master <= Récupérer la branche "master"
      • git push origin master <= Envoyer vos commits vers le serveur sur la branche "master"
    • Vous pouvez changer de branche de la même manière que le TP
      • Si vous avez une erreur remote branch does not exist : Essayez avec un checkout (exemple : git checkout -b coucou).

C'est quoi Gradle ?

Gradle est un outil de compilation, initialement dédié aux projets Java. Il se différencie de Ant (vu en TP2), car celui-ci gère également les dépendances de votre projet Java.

Ce type d'outil (comme Maven) est très utile, notamment lorsque les projets commencent à grossir, ou ont besoin de librairies : On ne stocke plus les libs avec le code, on les télécharges au besoin. Cela simplifie d'autant la gestion des versions de ces dernières.
